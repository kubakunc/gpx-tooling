# Phase 4.5: UX / Usability Pass

> REQUIRED: subagent-driven-development. Driven by the UX review of the Phase 4 app. Keep the ≥90% coverage gate green; add tests for any new pure helper.

**Goal:** Fix the P1 and high-value P2 usability problems found in the UX review so the app is honest, understandable, and friction-free for a real cyclist/runner. No new features — usability of what exists.

**References:** current screens under `src/routes/(tabs)/`; components in `src/lib/components/`; stores in `src/lib/stores/`.

---

## Fixes

### F1 (P1) — Honest state for not-yet-wired tools (convert, elevation, compare)
These show fictional data and their action buttons silently do nothing. Until wired (Convert+Elevation = Phase 5, Compare = Phase 7):
- Add a small non-intrusive **"Preview — not functional yet"** badge/banner at the top of `tools/{convert,elevation,compare}/+page.svelte` (subtle, uses `ink-faint`).
- Make each primary action button call `showToast('This tool isn't available yet', 'info')` instead of doing nothing.
- On the **hub**, add a tiny "Soon" pill to the Convert, Elevation fix, and Compare tiles (small, top-right of the tile) so users know before tapping. (Phase 5/7 will remove the pill + banner as each is wired.)

### F2 (P1) — Fix the misleading "Export from Strava & Komoot" related-app card
`(tabs)/+page.svelte`: it looks like an external app launch (↗) but navigates to `/files`. Make it honest: tapping shows `showToast('Strava & Komoot import is coming soon', 'info')` and does NOT navigate. Keep the styling; it's fine to keep the card as a "coming soon" teaser.

### F3 (P1) — Loading states for import/export
In `tools/{merge,trim,reduce}`, when `busy` is true: show a spinner + "Working…" on the active button (import/export) and disable it (already disabled — add the visible spinner). A simple inline SVG/CSS spinner is fine. Ensure the user never sees a frozen-looking screen during parse/serialize/share.

### F4 (P1) — Visible, switchable active file in Trim & Reduce
Today Trim/Reduce silently use `loadedFiles[0]`. Add a tappable **active-file selector**: a chip/row showing the current file name with a ▾ that opens a simple list of loaded files to switch (write the chosen id to `editSession.fileId`). If only one file, still show its name (no switcher needed). If none, keep the existing import empty state. Make it obvious which track is being edited and where it came from.

### F5 (P2) — Reduce: clearer framing, drop epsilon jargon
`tools/reduce`: replace "Accuracy 73% · High" + "ε=50m/ε=0m" sub-labels with an outcome-framed control. Add a tested pure helper `simplificationLabel(percent: number): 'Low'|'Medium'|'High'` in `src/lib/domain/usecases/reduceMapping.ts` (e.g. <34 Low, <67 Medium, else High — define exact thresholds + test). Header reads e.g. "Detail kept · High". Sub-labels become "Smaller file" / "More detail" (no meters). Keep the live Points/Size before→after readout (it's the real feedback). Default the slider to a sensible Medium (~50%) with a one-line helper.

### F6 (P2) — Export filenames derived from the source
Add a tested pure helper `exportName(sourceName: string, suffix: string, ext = 'gpx'): string` in `src/lib/domain/usecases/format.ts` that turns `morning.gpx` + `trimmed` → `morning-trimmed.gpx` (strip existing ext, append `-suffix.ext`; fall back to `suffix.ext` if no source). Use it in merge (`merged`), trim (`<src>-trimmed`), reduce (`<src>-reduced`). Add a couple of unit tests.

### F7 (P2) — Trim: couple the two sliders + clearer labels
`tools/trim`: prevent Start dragging past End and vice-versa by routing both through `editSession.setStartRatio/setEndRatio` (which already enforce `start<end` with MIN_GAP). Relabel the three boxes to be unambiguous: **"Cut before" / "Keep" / "Cut after"** (instead of Start/Kept/End). Keep the values.

### F8 (P2) — Wire the Files screen to loadedFiles
`(tabs)/files/+page.svelte`: replace the "Coming soon" placeholder with a real list of currently loaded files (name + distance/gain/duration via stats) and a remove (✕) action per row, plus an "Import GPX files" button. Empty state when none. This makes the bottom-nav "Files" destination useful and gives users a home for their imported tracks. Reuse `loadedFiles` store + `FileService`.

### F9 (P3) — Merge order clarity
`tools/merge`: add a one-line note under the file list: "Files are stitched together by timestamp." so the ↑↓ reorder behavior isn't confusing for timestamped tracks.

### F10 (P3) — Touch targets + undo on remove
Merge row buttons: bump up/down/remove to ≥40px and add spacing; separate the destructive remove (✕) from the reorder pair. On remove, show an undo toast ("Removed <name>" with an Undo action that re-adds it) — if the toast component supports an action; otherwise just confirm-on-remove is unnecessary, a plain toast is fine.
