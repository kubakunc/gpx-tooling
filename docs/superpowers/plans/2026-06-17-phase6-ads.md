# Phase 6: AdManager (AdMob + UMP consent, banner, interstitial)

> REQUIRED: subagent-driven-development + TDD. ≥90% coverage gate stays green (add `src/lib/ads/**` to coverage include). Three-stage review after.

**Goal:** Real ads via `@capacitor-community/admob` v6: UMP/GDPR consent on first launch → AdMob init → bottom **adaptive banner** on every screen → **interstitial** preloaded on edit, shown after a successful export with a 3-minute cap. Free, ad-supported, no ad-free path. **Google test ad IDs** (real IDs swapped later).

**Scope:** Native-only ads; on web/preview everything is a safe no-op and the existing placeholder banner shows. Use dependency injection so AdManager logic is unit-tested with a mocked plugin + injected clock.

**Plugin API (v6, verified):** `AdMob.initialize({ initializeForTesting:true })`; `AdMob.requestConsentInfo()` → `{ status: AdmobConsentStatus, isConsentFormAvailable }` (statuses `NOT_REQUIRED|OBTAINED|REQUIRED`); if `REQUIRED && isConsentFormAvailable` → `AdMob.showConsentForm()`. `AdMob.showBanner({ adId, adSize: BannerAdSize.ADAPTIVE_BANNER, position: BannerAdPosition.BOTTOM_CENTER, margin:0, isTesting:true })`. `AdMob.prepareInterstitial({ adId, isTesting:true })` / `AdMob.showInterstitial()`; events `InterstitialAdPluginEvents.{Loaded,Dismissed,FailedToLoad}`.

### Task 2: AdManager (TDD with mocked plugin + injected clock)
- `src/lib/ads/AdManager.ts`: a class with injectable deps `{ ads: AdLike; isNative: () => boolean; now: () => number }` and a small `AdLike` interface (subset: initialize, requestConsentInfo, showConsentForm, showBanner, prepareInterstitial, showInterstitial, addListener) so tests pass a fake. A module-level singleton `adManager` wires the real `AdMob`, `Capacitor.isNativePlatform`, `Date.now` for the app. Also expose a small svelte `writable` for `{ consent, bannerShown, ready }` if useful for UI.
  - `init()`: if `!isNative()` → no-op (set a "web" flag). Else: `requestConsentInfo()`; if `REQUIRED && isConsentFormAvailable` → `showConsentForm()`; then `initialize({initializeForTesting:true})`; then `showBanner(...)`. Persist consent obtained to the `settings` store. Wrap in try/catch (never throw to caller).
  - `prepareInterstitial()`: native-only; call `prepareInterstitial({adId,isTesting:true})`; on `Loaded` set `ready=true`, on `Dismissed` set `ready=false` + re-prepare, on `FailedToLoad` set `ready=false`. Idempotent (don't double-prepare).
  - `showInterstitialIfReady(onDismissed: () => void)`: if `!isNative()` or `!ready` or `!canShow(lastShown, now())` → call `onDismissed()` immediately and return. Else `showInterstitial()`, set `lastShown=now()`, and call `onDismissed()` on the Dismissed event (fallback: call it after show resolves). Never show mid-edit (caller controls timing).
  - `reopenConsentForm()`: native-only `showConsentForm()`/privacy options — for the Settings screen.
- TDD `AdManager.test.ts` with a fake `AdLike` + injected `now`: web → all no-ops, `showInterstitialIfReady` calls onDismissed immediately; native consent REQUIRED → calls showConsentForm then initialize then showBanner; interstitial gating respects `ready` and the 3-min cap (advance injected clock); Dismissed re-prepares.
- Commit `feat: add AdManager (consent, banner, interstitial, cap) + tests`.
